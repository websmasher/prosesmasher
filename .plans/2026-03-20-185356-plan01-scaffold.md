# Plan 01 Implementation: Project Scaffold

## Goal
Create parser crate, add workspace deps, write all domain types, port traits, stubs. Everything compiles with `cargo check`. No business logic.

## Approach

### Implementation (7 tasks, parallelizable in 3 waves)

**Wave 1 — Types & Crate Creation (parallel)**
1. `domain/types/src/` — split into locale.rs, document.rs, metadata.rs, config.rs, error.rs, update lib.rs
2. Create `adapters/outbound/parser/` crate stub with Cargo.toml + lib.rs

**Wave 2 — Dependencies & Ports (after wave 1)**
3. Update workspace Cargo.toml: add parser to members, add all workspace deps
4. Update guardrail3.toml: add parser crate entry
5. `ports/outbound/traits/src/` — split into file_reader.rs, document_parser.rs, config_loader.rs, update lib.rs

**Wave 3 — Wire everything (after wave 2)**
6. Update adapter stubs (fs, parser, cli) with real imports
7. Update app/core stub

### Verification
```bash
cargo check
cargo clippy --workspace --all-targets --all-features -- -D warnings
cargo test --workspace
```

### Test Strategy for Plan 01

Plan 01 is types-only — no logic to test. But we set up foundations:

1. **Compile-time tests** — `cargo check` + `cargo test --workspace` passing IS the test
2. **Default trait tests** — `DocumentMetadata::default()` and `HeadingCounts::default()` should produce zeroed structs. Add `#[test]` in domain/types.
3. **Error Display tests** — each error variant's Display impl should produce meaningful strings. Add `#[test]` in domain/types.
4. **Locale exhaustiveness** — test that all 7 locales are distinct values (coverage for Copy + Eq derives)
5. **Range construction** — test Range { min, max } with Copy derive works

These are cheap compile-verification tests but they lock in the type contracts early.
