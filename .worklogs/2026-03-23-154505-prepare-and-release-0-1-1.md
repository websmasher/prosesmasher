## Summary
Cleaned up the remaining release blockers after the preset-embedding fix: added missing crates.io metadata/readme coverage for internal publishable crates, fixed strict clippy/MSRV issues, and bumped the full `prosesmasher` crate graph to `0.1.1` so a corrected release can go out without overwriting the broken `0.1.0`.

## Context & Problem
After fixing the installed-CLI preset packaging bug, the repo still was not in “release clean” shape:

- `cargo clippy -q --all-targets --all-features` failed because several internal publishable crates lacked `readme`, `keywords`, and `categories`
- one public helper and one public CLI entrypoint were missing pedantic `# Errors` docs
- `Sentence::word_count()` was marked `const`, which violates the workspace MSRV lint
- `0.1.0` was already published and broken on crates.io, so the fix needed a coordinated version bump rather than another dry-run against the old version

The user asked to “fix everything fixable on this repo and release the new version,” so the release surface had to be clean, not just the local runtime behavior.

## Decisions Made

### Add real workspace-level README coverage instead of suppressing metadata linting
- **Chose:** Add `apps/prosesmasher/README.md` and point internal publishable crates at it with explicit `readme` paths, plus `keywords` and `categories`.
- **Why:** The crates are published to crates.io and should have minimally coherent metadata rather than lint suppressions.
- **Alternatives considered:**
  - Allow `clippy::cargo_common_metadata` — rejected because the release issue was real, not just stylistic.
  - Add separate README files for every internal crate — rejected because it would add noise without real user benefit right now.

### Keep public helper functions documented instead of hidden
- **Chose:** Add proper `# Errors` docs to `parse_config_json()` and the CLI `run()` entrypoint.
- **Why:** Those functions are part of the published crate API surface, and pedantic clippy is intentionally enabled.
- **Alternatives considered:**
  - Add `#[allow(clippy::missing_errors_doc)]` — rejected because the API is small and easy to document properly.

### Remove the unnecessary `const` on `Sentence::word_count()`
- **Chose:** Make `word_count()` a normal function.
- **Why:** There is no meaningful need for `const` evaluation here, and the current workspace MSRV is lower than the stabilization point clippy enforces.
- **Alternatives considered:**
  - Raise the workspace MSRV — rejected because this was a single over-eager `const`, not a reason to increase the toolchain floor.
  - Suppress `clippy::incompatible_msrv` — rejected because the lint was correct.

### Bump the entire crate graph to `0.1.1`
- **Chose:** Move all `prosesmasher*` crates from `0.1.0` to `0.1.1` in workspace metadata and internal dependency constraints.
- **Why:** `0.1.0` is already on crates.io and contains the broken preset workflow; a fixed release must be a new version across the whole graph.
- **Alternatives considered:**
  - Reuse `0.1.0` locally and rely only on CI metadata — rejected because crates.io publication requires a new version.

## Architectural Notes
This change does not alter the validation architecture. It tightens the published crate graph:

- internal crates now carry enough metadata to satisfy strict cargo/clippy checks
- API docs match the workspace’s pedantic standards
- the release graph is version-coherent at `0.1.1`

The earlier preset embedding change remains the functional fix; this work is the release hardening required to publish it confidently.
