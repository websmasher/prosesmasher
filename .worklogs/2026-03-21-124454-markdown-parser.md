# Plan 02: markdown.rs + bug fixes + 5 adversarial fixtures

## Summary

Implemented `markdown.rs` — pulldown-cmark event walker that builds the full Document tree. Created 5 adversarial fixture files (~25k words total) for integration testing. Fixed 5 bugs found by 3 rounds of test attacks. Wired `lib.rs` to expose the parser module.

## Decisions

- **H1 and H2 both create sections**: H1 mid-document is valid markdown and should be faithfully parsed. Checks layer (Plan 04+) enforces structure rules.
- **Nested lists use a stack**: `list_stack: Vec<ListState>` replaces flat fields. Inner lists become separate `Block::List` entries in the section, not nested inside outer ListBlock items. Known limitation: outer item text before inner list is lost due to `list_item_text` not being stacked.
- **Link text accumulates before context-specific returns**: `link_text.push_str` runs before `in_heading`/`in_list_item` early returns, so link display text is preserved. Structured `Link` objects are only pushed to `para_links` (headings and list items don't have link fields in the domain model).
- **Fixture expected counts from parser output**: Agent-generated counts were wrong. Parser output is ground truth, manually verified for unit tests, fixture counts pinned from actual runs.

## Bugs fixed

1. H1 mid-document silently dropped → now creates sections (`level <= 2`)
2. H1 mid-document doesn't create section → same fix
3. Nested lists flatten → list stack
4. Links inside headings have empty text → link_text before heading return
5. Links inside list items dropped → link_text before list_item return
