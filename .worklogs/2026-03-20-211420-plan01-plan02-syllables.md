# Plan 01 scaffold + Plan 02 syllables.rs

## Summary

Implemented Plan 01 (full project scaffold with domain types, port traits, parser crate) and the first piece of Plan 02 (syllable counting module with hyphenation dictionary integration). Hardened tests through 2 rounds of adversarial review. Fixed bugs found by tests. Fixed medium-severity code review findings.

## Decisions

- **Accented Latin vowels**: Added all common accented forms (é, ü, ö, ä, etc.) to `is_vowel`. Initially missing — caught by adversarial test review.
- **Syllable fallback strategy**: When dictionary returns >1 segments, take `max(dictionary, vowel_count)` to compensate for under-segmenting. When dictionary returns 1 segment, fall back to vowels if they find >1 cluster.
- **Silent-e limitation accepted**: Words like "fire", "smile" are over-counted as 2 syllables. Can't distinguish "dictionary knows it's 1" from "dictionary doesn't know the word." Acceptable for readability scores which are statistical.
- **`Range` validation**: Made fields private, added `Range::new(min, max) -> Option<Self>` that rejects `min > max`.
- **`Sentence.word_count` removed**: Was a separate field that could diverge from `words.len()`. Replaced with `word_count()` method.
- **Test organization**: R-TEST-09 compliant — tests in sibling `_tests.rs` files, not inline.
- **Dictionary caching deferred**: `Standard::from_embedded()` called per-word currently. Will cache in `MarkdownParser` struct when building the adapter.
