# Plan 02: segmenter.rs implementation

## Summary

Implemented `segmenter.rs` — ICU4X-based sentence/word segmentation with syllable counting. Two rounds of adversarial test attack hardened the test suite from 24 to 31 tests, eliminating silent-pass patterns, tautological assertions, and hedging.

## Decisions

- **ICU4X is locale-independent for segmentation**: `SentenceSegmenter::new()` and `WordSegmenter::new_auto()` don't take locale. Locale only flows to `count_syllables`. This is fine — ICU4X handles multilingual text without hints.
- **Pinned ICU4X behavior**: Probed actual ICU4X output for abbreviations ("Dr." splits), newlines (split), em-dashes (word boundary), contractions ("don't" = 1 word), hyphens ("well-known" = 2 words). Tests pin these exact values instead of hedging.
- **`assert_sentences` helper**: All tests use this helper that asserts sentence count upfront, eliminating silent-pass risk from `let Some else return` patterns.
- **`string_slice` allows**: Two `#[allow(clippy::string_slice)]` for ICU4X break point indexing — ICU4X guarantees UTF-8-valid boundaries.
- **Locale pass-through gap accepted**: All locale test words happen to produce the same syllable count via vowel fallback as via the correct dictionary. Can't prove locale pass-through without mocking. Accepted as test limitation — code is correct, compiler enforces types.
