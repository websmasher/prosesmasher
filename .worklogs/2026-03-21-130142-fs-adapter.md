# Plan 03: FS adapter + config loading

## Summary

Implemented FsFileReader (FileReader trait) and FsConfigLoader (ConfigLoader trait). JSON config DTOs with serde deserialization and garde validation, separate from domain types. DTO → domain conversion with locale parsing and Range validation.

## Decisions

- **Serde DTOs in adapter layer**: ConfigDto, TermListsDto, ThresholdsDto, RangeDto live in `config_dto.rs` in the FS adapter. Domain types remain pure (no serde).
- **camelCase JSON → snake_case Rust**: via `#[serde(rename_all = "camelCase")]`
- **simplicityPairs**: JSON `[["complex", "simple"]]` → `Vec<SimplePair>` via `Vec<[String; 2]>` DTO
- **Garde validation**: locale length, dive into nested DTOs. Range min<=max validated during DTO→domain conversion.
- **clippy.toml exemptions**: `#[allow(clippy::disallowed_methods)]` on fs read and serde_json::from_str with reason comments. This IS the centralized fs module.
