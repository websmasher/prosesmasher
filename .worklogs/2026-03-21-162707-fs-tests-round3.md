# FS adapter: 3 rounds of test attacks, converged

## Summary

Hardened FS adapter tests through 3 rounds of adversarial attack. 22 → 52 tests. Fixed race condition in temp file helper, eliminated all `unwrap_or_default` in test paths, added garde detection via error message content, fixed ReadError mapping in config_loader. Round 3 found no new issues — converged.

## Bugs found and fixed

- Config loader mapped ALL ReadError variants to ConfigError::NotFound (PermissionDenied and Io errors reported as "not found"). Fixed to include descriptive message.
- Temp file helper used shared filename causing race conditions in parallel tests. Fixed with atomic counter.

## Key test improvements

- Sentinel-per-field swap detection for all 28 fields
- Garde vs parse_locale distinguished by error message content
- Helpers panic on unexpected results (no silent pass)
- File reader: empty file, directory, trait object, not-NotFound for directories
- Edge cases: BOM, trailing comma, null values, negative numbers, wrong types
