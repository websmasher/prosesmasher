# Initial scaffold + check structure plan

## Summary

Set up prosesmasher as a Rust CLI project with hexagonal architecture (guardrail3-generated). Wrote the check structure plan defining 33 prose quality checks across 4 groups.

## Decisions

- **ICU4X** for word/sentence segmentation (not `unicode-segmentation`) — locale-aware, handles abbreviations per language, covers all 7 target languages (en/ru/de/es/pt/fr/id).
- **`hyphenation`** for syllable counting (readability scores) — Knuth-Liang TeX patterns, multilingual.
- **Hand-rolled readability formulas** — existing crates are GPL (`rust_readability`) or English-only (`kincaid`). Each formula is 2-3 lines of arithmetic.
- **No regex** — all pattern detection uses token matching on ICU4X-segmented words/sentences.
- **No LLM** — all checks deterministic. False negatives acceptable, false positives not.
- **Per-locale config** — same detection mechanisms, different term/phrase lists per language.
