# JSON output format for CI/automation

## Summary

Added `--format json` flag to `prosesmasher check`. Outputs structured JSON with file path, overall success, counts, and per-check results including observed values. Default remains `--format text`.

## JSON schema

```json
{
  "file": "article.md",
  "success": false,
  "evaluated": 23,
  "passed": 20,
  "failed": 3,
  "checks": [
    {
      "id": "banned-words",
      "label": "Banned Words",
      "success": false,
      "observed": ["actually"]
    }
  ]
}
```

## Key files

- `args.rs` — `OutputFormat` enum (Text/Json), `--format` flag with default "text"
- `output.rs` — `FileResult`/`CheckOutput` structs with serde Serialize, `build_file_result` for testing
- `main.rs` — passes format through to `output_result`
