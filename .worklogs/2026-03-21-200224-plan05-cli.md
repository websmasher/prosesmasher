# Plan 05: CLI adapter — end-to-end working

## Summary

Implemented the clap-based CLI with `prosesmasher check` command. Composition root wires all adapters: FsFileReader + FsConfigLoader + MarkdownParser → parse → run all 32 checks → print results. Supports single file or recursive directory scanning.

## Usage

```bash
prosesmasher check article.md --config config.json
prosesmasher check docs/ --config config.json
prosesmasher check article.md --config config.json --group terms
```

## Key files

- `crates/adapters/inbound/cli/src/args.rs` — clap argument parsing
- `crates/adapters/inbound/cli/src/checks.rs` — check collection with group filtering
- `crates/adapters/inbound/cli/src/output.rs` — result formatting
- `crates/adapters/inbound/cli/src/main.rs` — composition root
- `tests/fixtures/test-essay.md` — end-to-end test fixture (has em-dashes, banned word, title case)
- `tests/fixtures/test-config.json` — test config

## Decisions

- No std::process::exit (banned). Return Err from main for non-zero exit.
- No colored output yet (plain text, can add later).
- walkdir for recursive .md discovery in directories.
- Group filtering: --group terms|patterns|structure|readability.
- Unknown group returns error with valid group list.
